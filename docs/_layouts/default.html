Challenge #1:

How to design an algorithm to display appointments in a calendar by month?



Solution:
  • Add a VBox to each grid inside the GridPane to properly align ComboBox used for appointment lists and label used for displaying each date in the selected month.
  • Download all appointments from the database and map them by months then by dates in a month (2-layer map). Iterate over the map by dates to add lists of appointments to ComboBoxes.

Challenge #2:

How to design an algorithm to display appointments by week with overlapped time between grids?



Solution:
  • Anchor Labels with appointment details to AnchorPanes added to GridPane. The number of rowspan is determined based on the number of appointment hours.
  • For appointments starting/ending at non-zero minutes, the top and bottom anchor values for the Label are set based on the startMin and endMin

Challenge #3:

How to allow sharing one instance of the AppointmentDB class so that controllers can access lists and maps of appointments downloaded from the database without having to access the database?

Solution:

Singleton design pattern was used for the AppointmentDB class, which allows only one instance of the AppointmentDB class to be created in memory and shared among multiple classes as the lists/maps of appointments are updated/downloaded from the DB after each INSERT, DELETE, UPDATE query.

Challenge #5:

The displayScene() method in the WindowsDisplay class is used to load FXML, set Scene, and show Stage. Beside the required parameters for loading FXML and setting Stage (FXMLPath and title), there are certain scenes that required optional parameters (ResourceBundle for loading language files, ownerStage for modal Windows, and Customer and/or Appointment to set the selected Customer and/or Service in the controller data manipulation. How can this be achieved without making the optional parameters required?

Solution:

The Builder design pattern (WindowsBuilder class) was used to allow the necessary parameters to be passed to the Builder object through method chaining via setters in the WindowsBuilder class. A WindowsDisplay object is generated with a final build() call.

Challenge #6:

How to allow users to search the Customer List?

Solution:
  • Wrap a FilteredList around the ObservableList.
  • Add a ChangeListener to the search TextArea. Whenever the TextArea is changed, the Predicate of the FIlteredList as defined by the lambda expression is updated. The Predicate matches the element in the ObservableList that will be visible.
  • Finally, wrap the FilteredList inside a SortedList for sorting purpose and bind the sorted list to the TableView.
